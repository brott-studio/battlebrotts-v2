## 4. Acceptance criteria — per design §8

### ✅ AC-1 — Fresh user: overlay appears on first combat

Verified by the test suite's `_test_ac1_overlay_spawns_on_fresh_first_run`: with a cleared store, instantiating `main.tscn` produces an `EnergyExplainerOverlay` Panel child under `_ui_layer` with a `Body` Label carrying `ENERGY_EXPLAINER_TEXT` (`⚡ The blue bar is your Energy — it powers your weapons and regenerates over time.`), a `▲` arrow Label, and a `GotItButton`. Corroborated by the `overlay_present.png` screenshot in §6 — the Panel renders top-left at (16, 72) as specified in the design.

### ✅ AC-2 — After dismissal, overlay never reappears

Verified by `_test_ac2_overlay_skipped_when_marked_seen`: after calling `FirstRunState.mark_seen("energy_explainer")` and re-instantiating `main.tscn`, `get_node_or_null("EnergyExplainerOverlay")` returns `null`. The `user://first_run.cfg` ConfigFile contains `[seen] energy_explainer=true`. Corroborated by the `overlay_absent.png` screenshot in §6.

### ✅ AC-3 — Auto-dismiss via input

Verified by `_test_ac3_input_dismisses_and_persists`: a synthesized `InputEventKey` (pressed=true) routed to `_unhandled_input` dismisses the overlay AND persists the `energy_explainer=true` flag. The implementation intentionally uses `_unhandled_input` (not `_input`) so that the "Got it!" button's own press event is not pre-empted when AC-1 dismissal is taken via the button. The fall-through comment confirms same-frame gameplay input (e.g. `KEY_1` speed toggle) still applies — correct design.

### ✅ AC-4 — Auto-dismiss via tick budget

Verified by `_test_ac4_tick_budget_dismisses_and_persists`: after `ENERGY_EXPLAINER_TICK_BUDGET` (60) `_process` frames with no input, the overlay frees itself and the persistence flag flips. 60 ticks ≈ 1 s @ 60 fps matches design §3.

### ✅ AC-5 — S17.1-003 legend unmodified

**Bit-exact pixel-identity confirmed** over the strict legend region defined by `_setup_energy_legend` offsets (`offset_left=20, offset_top=42, offset_right=520, offset_bottom=62`), comparing `overlay_present.png` vs `overlay_absent.png` pixel-by-pixel:

```
strict legend-region byte diffs: 0 of 40000
pixel-identical (strict region): True
```

Zero bytes of difference across the 500×20 legend row. The overlay Panel's `offset_top=72` is intentionally below the legend's `offset_bottom=62`, leaving a 10 px gap; no z-order contention. Legend node (`EnergyLegend` Label under `_ui_layer`) is not reparented, mutated, or re-ordered by the S17.1-004 diff — confirmed by `git diff origin/main..pr168 -- godot/main.gd` (adds only; `_setup_energy_legend` body is byte-identical).

Full-image diff (3.4%) is accounted for entirely by the overlay Panel itself (expected).

### ✅ AC-6 — Shared persistence API — reserved `crate_first_run` key

Verified by `_test_ac6_shared_persistence_api`: `FirstRunState.has_seen("crate_first_run")` returns `false` on fresh save and `true` after `FirstRunState.mark_seen("crate_first_run")`. No code change in `first_run_state.gd` required — `SECTION = "seen"` plus flat `get_value` / `set_value` already accept arbitrary string keys. S17.1-006 can adopt the helper unchanged.

## 7. Non-blocking notes

- **Autoload guard is double-belt-and-suspenders.** `_maybe_spawn_energy_explainer` checks both `Engine.has_singleton("FirstRunState")` and `get_node_or_null("/root/FirstRunState")` before spawning. Either one is sufficient for autoloads; harmless redundancy.
- **Arrow offset is a magic number.** `arrow.offset_left = 90.0` is hand-tuned to land under the ⚡ glyph at the legend's default font size. If S17.1-003's legend font size ever changes, the arrow will drift. Called out in the design doc §4.2 — acceptable for now; a future ticket could compute it from the legend's glyph rect.
- **ObjectDB leak warnings at test exit.** Present in both S17.1-003 and S17.1-004 suites; not introduced by this PR. Harmless for CI exit code.

None of the above block the Pass verdict.
