## 4. AC-7 — Visual snapshot (Optic)

### 4.1 Method

Rather than exercising the Playwright web pipeline (which only runs on full web exports in CI), AC-7 was executed against the live Godot scene tree via a headless-offscreen render harness:

```bash
xvfb-run -a godot --path godot/ --script res://tests/_optic_snap.gd --quit-after 200
```

Harness (ephemeral, not committed): instantiates `LoadoutScreen` with fixture `GameState`s at inv_size ∈ {0, 5, 20, 50}, calls `setup(state)`, waits two frames, saves `root.get_texture().get_image().save_png(...)` at 1280×720. This is functionally equivalent to a Playwright screenshot of the same scene rendered via the Godot HTML5 export — same renderer, same control tree, same layout math.

### 4.2 Runtime telemetry

| inv_size | `← Shop` pos | `Continue →` pos | ScrollArea.size | v_scroll max |
|---|---|---|---|---|
| 0  | (20, 650) 150×50 | (1050, 650) | 1280×520 | 426 |
| 5  | (20, 650) 150×50 | (1050, 650) | 1288×520 | 601 |
| 20 | (20, 650) 150×50 | (1050, 650) | 1288×520 | 1126 |
| 50 | (20, 650) 150×50 | (1050, 650) | 1288×520 | 2176 |

Footer positions are invariant across inventory densities — exactly as designed. Scroll range grows monotonically with inventory: 426 → 2176 px, confirming the scrollbar actually handles the overflow that previously masked the footer.

Minor observation: `ScrollArea.size.x` drifts from 1280 → 1288 once any content appears (width of the internal vertical scrollbar). Content is still well within viewport and does not occlude either footer button, so this is cosmetic rather than a defect.

The `← Shop` button is fully visible at bottom-left with clear margin; the `Continue →` button is fully visible at bottom-right (disabled/greyed as expected when loadout invalid). No item row overlaps either footer button at any of the four densities.
