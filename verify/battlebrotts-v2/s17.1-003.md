### Minor follow-ups (informational, not blocking)

- `⚡` glyph renders via system font fallback in the headless capture; on the web export this will depend on the bundled font's glyph coverage. Not observed as broken, but worth a confirm pass in the next playtest build. If the glyph ever renders as a missing-box tofu, swap for a text prefix like `[⚡] ` or drop the emoji entirely — the anchor-text unit assertion is tolerant of that change.
- Shop card rendering was exercised via unit test + source diff rather than direct viewport capture because the Shop screen's entry state is gated behind the crate/mystery interaction. If future verify sprints want a direct Shop card snapshot, the harness would need to either drive the `_toggle_expand` input or seed a post-gate state. Not blocking for this task.
