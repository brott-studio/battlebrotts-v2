## 5. Visual verification (scene-graph + runtime state)

Headless Godot's dummy renderer does not provide a viewport texture for PNG capture. Visual verification was therefore done by instantiating the packed `trick_choice_modal.tscn` at runtime and asserting on the live scene graph and label state — the equivalent single source of truth that the actual pixel would reflect.

**First-run scenario** (via dedicated screenshot harness `tests/_screenshot_s17_1_006.gd`, runtime-captured state):

```
[first-run] framing visible=true text=Crates are optional loot. Opening might give you an item — or nothing.
```

- `FirstRunFraming` node exists as the **first child** of `$Overlay/Panel/VBox` (confirmed above `TopRow` per design §4.2).
- `visible = true` after `show_trick({id: "crate_find", ...})`.
- `modulate = Color(0.75, 0.75, 0.75, 1)` (dimmed context-chrome per §4.2).
- `theme_override_font_sizes/font_size = 13` (one step smaller than Prompt per §4.2).
- `autowrap_mode = 3` (WORD_SMART per §6 edge case 5).
- `horizontal_alignment = 1` (centered per §4.2).
- Text rendered verbatim with the em-dash (U+2014).

**Second-run scenario** (same harness, second modal instance after the first):

```
[second-run] framing visible=false
```

- After the first `show_trick` fired `mark_seen("crate_first_run")`, a second modal instance with the same `crate_find` trick renders with `_first_run_framing.visible = false`. Modal otherwise identical to main-line behavior.

**Non-crate isolation** (AC-3, from unit suite): after calling `show_trick({id: "risk_for_reward", ...})` with `crate_first_run` unseen, the framing stays hidden **and** `FirstRunState.has_seen("crate_first_run")` remains `false`. No stray mark.
